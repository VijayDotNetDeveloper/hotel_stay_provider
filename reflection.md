# Reflection

- I used GitHub Copilot instructions and explicit prompt documentation to align the implementation with the spec, especially when converting the backend from controller-based APIs to Minimal API routes.
- I captured prompt wording and decision rationale in `prompts.md` to make AI-assisted decisions visible and traceable.
- Improvements made include adding frontend component tests with Vitest, moving cancellation policy values to spec-aligned constants, and creating a visible sort control in the search results.
- With more time, I would add end-to-end UI tests, a coverage report, and more granular prompt/response logging for each development step.
- I also identified that AI usage should be documented across analysis, modelling, implementation, and testing, not just code generation, and I have started that with the updated prompt/reflection notes.
